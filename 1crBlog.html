How to set up your workspace on our LAMP Droplet

This blog post will describe how to personalize your workspace to be a little more usable on our shared LAMP Droplet. This tutorial does not cover everything, just things that I have personally done to make my workspace better.

To follow this tutorial, you should have already logged into the Droplet at 64.227.19.180 through either PuTTY or SSH with your account for the server.

Step 1: Change your password

Before you do anything else, you should change your password so someone can't impersonate you using the publicly accessible password for your account. In the terminal, type:

$ passwd

It will ask for your current password, then it will ask you to type in a new password and then retype the same password. Choose a password that is different from your other passwords, as this server is not particularly secure!

Step 2: Create a directory for your HTML documents

First, you need to move to the html folder. Type:

$ cd ../../var/www/html

This command simply moves you back two folders and then moves you into var, then www, then finally html. To create a new folder, use the proj.sh script already available to you.

$ sudo proj.sh

Now, rename the folder.

$ sudo mv new-project <your username/whatever folder name you'd like>

Step 3: Make that directory yours

$ ls -la

Notice under owner and group for the folder you just made it says root for both? That's because you used root to get permission to create the folders using proj.sh. To make it easier to work with, change the owner and group to your account by typing:

$ sudo -R chown <your username>:<your username> <whatever you renamed the folder to>

This command recursively (-R) changes the owner and group (owner:group) of the folder specified. This allows you to do things like making directories without using the sudo command.

(Optional) Step 4: Personalize your VIM profile

Nano has been recommended for this class because it displays the commands on screen. If you want something more powerful, I recommend VIM. There are a few things you will need to understand and a few commands you will need to memorize. Navigate into the folder you created and then type:

$ vim index.html

Now you are in VIM. Use the arrow keys to navigate the file. However, you won't be able to edit it yet! That is because you are "command mode" or "normal mode." In the bottom left, you will see no text appear. In this mode, you can type : to begin a command. :q will quit VIM, :w will save changes, :wq will save your changes and quit, and :q! will quit without saving any changes.

To edit the file, you must enter "insert mode." While in "command mode" hit i. Now, in the bottom left, you will see "-- INSERT --" which tells you that you are in insert mode. Now you can type into the file. To go back to "command mode" hit the escape key. While in insert mode, you won't be able to do any commands.

This is all you need to know how to use VIM! Now to customize it. :q out of the file. Now go in to your VIM profile by typing:

vim ~/.vimrc

In this file, you can store all your settings for VIM that apply every time you use it. Here are my suggested ones:

    
			
  • set tabstop=4
    • 
			
  • set autoindent
    • 
			
  • syntax on
    • 
			
  • colorscheme slate
    •

Tabstop determines the length of a tab. Autoindent mkes it so when you hit enter, it automatically indents to the indent of the previous line. Syntax turns on syntax highlighting. Colorscheme determines the color scheme of VIM, and there are mny to choose from. I suggest slate for HTML, however you can see all the schemes available to you by entering vim and typing :colorscheme (don't forget the space!) and then hitting tab multiple times.
